java.util.concurrent package

ExecutorService interface
    create an instance of ExecutorService
        Executors factory class
            - single-thread
                Executors.newSingleThreadExecutor()
                Executors.newSingleThreadScheduledExecutor()
            - pooled-thread - thread pool: a collection of pre-instantiated threads that can be reused to execute tasks
                Executors.newCachedThreadPool()
                Executors.newFixedThreadPool(int nThreads)
                Executors.newScheduledThreadPool(int corePoolSize)
                Executors.newVirtualThreadPerTaskExecutor()

    methods
        void execute(Runnable command)
        Future<?> submit(Runnable task)
        <T> Future<T> submit(Callable<T> task)
        <T> List<Future<T>> invokeAll(Collection<? extends Callable<T>> tasks)
        <T> T invokeAny(Collection<? extends Callable<T>> tasks)

    states
        Active -> shutdown() -> Shutting Down (rejected new tasks) -> all tasks finished -> Shutdown
            close() - automatically called in try-with-resources block (available since Java 19)
            shutdown()
            shutdownNow()

    ScheduledExecutorService interface extends ExecutorService
        create an instance
            Executors.newSingleThreadScheduledExecutor()
        methods
            schedule(Runnable command, long delay, TimeUnit unit)
            schedule(Callable<V> callable, long delay, TimeUnit unit)
            scheduleAtFixedRate(Runnable command, long initialDelay, long period, TimeUnit unit)
            scheduleWithFixedDelay(Runnable command, long initialDelay, long delay, TimeUnit unit

Future interface
    methods
        boolean isDone()
        boolean isCancelled()
        boolean cancel(boolean mayInterruptIfRunning)
        V get()
        V get(long timeout, TimeUnit unit)

    TimeUnit enum
        NANOSECONDS, MICROSECONDS, MILLISECONDS, SECONDS, MINUTES, HOURS, DAYS

concurrent collections: try to solve common memory consistency errors
    - memory consistency error: when two threads have inconsistent views of what should be the same data
    - use when multiple threads modify a collection outside a synchronized block
        - possible side effects
            - ConcurrentModificationException at runtime
            - corrupt data

    classes
        ConcurrentHashMap
        ConcurrentLinkedQueue
        ConcurrentSkipListMap - sorted concurrent collection
        ConcurrentSkipListSet - sorted concurrent collection
        CopyOnWriteArrayList - creates a copy of the collection any time it is modified
        CopyOnWriteArraySet - creates a copy of the collection any time it is modified
        LinkedBlockingQueue - provides overloaded versions of offer() and poll() with timeout

    synchronized Collections methods: to obtain synchronized versions of existing non-concurrent collection objects
        synchronizedCollection(Collection<T> c)
        synchronizedList(List<T> l)
        synchronizedSet(Set<T> s)
        synchronizedMap(Map<K,V> m)
        synchronizedNavigableSet(NavigableSet<T> s)
        synchronizedNavigableMap(NavigableMap<K,V> m)
        synchronizedSortedSet(SortedSet<T> s)
        synchronizedSortedMap(SortedMap<K,V> m)
