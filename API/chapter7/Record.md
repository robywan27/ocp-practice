## Immutable Classes
### Immutable Object Pattern
  - encapsulation: restricts access to members of a class -> declare instance variables as private
  - must not contain any mutator methods (setters)

## Records
A record defines an encapsulated and immutable class with additional compiler-generated boilerplate:

| What                                         | Description                                                           |
|----------------------------------------------|-----------------------------------------------------------------------|
| constructor                                  | params are in the same order as record declaration (long constructor) |
| accessor methods                             | one for each instance variable (no get/is prefix)                     |
| equals(), hashCode(), and toString() methods |                                                                       |

## Implicit modifiers
| What                      | Modifier            |
|---------------------------|---------------------|
| record                    | final               |
| record instance variables | private final       |

## Special syntax
| What                | Description                                                                                          | Syntax                                                                                                |
|---------------------|------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------|
| compact constructor | - no parameter list<br/>- executes the full constructor after passing manipulated input params to it | RecordName {<br/>&emsp; // optional validation guards<br/>&emsp; // input param transformations<br/>} |

## Rules
| MUST             |                                                                                                                                                                                                                             | CANNOT                |                                                                                 | CAN                |                                                                               |
|------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------|---------------------------------------------------------------------------------|--------------------|-------------------------------------------------------------------------------|
| long constructor | - have same parameter list as record declaration<br/>- initialize all instance variables<br/>- if overloaded constructors are present, the first line must be a call to another constructor (this(...)) or long constructor |                       |                                                                                 |                    |                                                                               |
|                  |                                                                                                                                                                                                                             | instance fields       | be modified after they are initialized in the constructor                       |                    |                                                                               |
|                  |                                                                                                                                                                                                                             |                       | be added outside the record declaration nor initialized outside constructor     |                    |
|                  |                                                                                                                                                                                                                             | a record              | be extended nor can inherit another record/class -> cannot be declared abstract |                    |                                                                               |
|                  |                                                                                                                                                                                                                             | compact constructor   | modify instance variables directly                                              |                    |                                                                               |
|                  |                                                                                                                                                                                                                             | instance initializers | be declared in a record                                                         |                    |                                                                               |
|                  |                                                                                                                                                                                                                             |                       |                                                                                 | a record           | have no fields                                                                |
|                  |                                                                                                                                                                                                                             |                       |                                                                                 |                    | implement interfaces                                                          |
|                  |                                                                                                                                                                                                                             |                       |                                                                                 |                    | support static fields, methods, and static initializers (no need to be final) |
|                  |                                                                                                                                                                                                                             |                       |                                                                                 |                    | support nested classes, interfaces, enums, and records                        |
|                  |                                                                                                                                                                                                                             |                       |                                                                                 |                    | support overriding methods (e.g. accessors, toString())                       |
|                  |                                                                                                                                                                                                                             |                       |                                                                                 | to modify a record | make a new object (copy) with modified fields                                 |
|                  |                                                                                                                                                                                                                             |                       |                                                                                 | the compiler       | omit generation of long constructors if an equivalent one is defined manually |

## Pattern matching
| How                           | Type                        | Special cases | Example                                                                                                                                                                                                                | Notes                                                                                                                                      |
|-------------------------------|-----------------------------|---------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------|
| if statements                 | simple variable match       |               | if (animal instanceof Monkey monkey)                                                                                                                                                                                   |                                                                                                                                            |
|                               | match members of the record |               | if (animal instanceof Monkey(String name, int age))                                                                                                                                                                    | - must include all fields in order<br/>- name of fields is irrelevant<br/>- type of the record and each field must match (at compile-time) |
|                               | nested record patterns      |               | if (c instanceof Couple(Bear b, Bear b2))<br/>if (c instanceof Couple(Bear(String name, List<String> f), Bear b))<br/>if (c instanceof Couple(Bear(String name, List<String> f), Bear(String name2, List<String> f2))) |                                                                                                                                            |
|                               |                             | with var      | if (c instanceof Couple(var b, Bear b2))<br/>if (c instanceof Couple(Bear(var name, List<String> f), var b))<br/>...                                                                                                   |
|                               |                             | generics      | if (c instanceof Couple(var b, Object b2))<br/>if (c instanceof Couple(var b, List b2))<br/>if (c instanceof Couple(var b, List<?> b2))<br/>...                                                                        |
| switch statements/expressions |                             |               | switch (animal) {<br/>&emsp;&emsp;case Monkey(String name, int age) -> ...<br/>&emsp;&emsp;case Lion l -> ...<br/>}                                                                                                    | - can omit default branch when all possible record types are covered<br/>- type matters for when clauses                                   |
